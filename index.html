Checkpoint 7: JavaScript

Checkpoint 7: JavaScript

1. ¿Qué diferencia a JavaScript de cualquier otro lenguaje de programación?

JavaScript se ejecuta principalmente en navegadores web, permitiendo interactividad sin instalar nada. Es interpretado, dinámico y soporta programación orientada a objetos, funcional y basada en eventos.

1. JavaScrit se está volviendo muy popular por diferentes razones

Es como un idioma universal para crear sitios web

Imagínate que JavaScript es como el inglés del mundo digital. Una de las principales razones por las que es tan popular es porque sirve para crear todo tipo de cosas en internet. Piensa en una tienda online: JavaScript puede crear tanto la parte que tú ves (el diseño bonito, los botones que haces clic, las animaciones) como la parte invisible que hace que todo funcione por detrás (como procesar tu pedido o calcular el precio total). Es como tener una herramienta que te sirve tanto para decorar tu casa como para arreglar la fontanería. Esta capacidad de hacer "todo en uno" es muy valiosa para las empresas, porque pueden usar una sola herramienta en lugar de varias diferentes.

Técnicamente: JavaScript es un lenguaje de pila completa (full-stack language) que puede utilizarse tanto para desarrollo front-end como back-end. Esto significa que puede manejar todo el proceso de desarrollo de software.

  • Front-end: La parte visible de una aplicación web (interfaces de usuario interactivas y dinámicas)
  • Back-end: La parte del servidor que maneja la lógica, bases de datos y procesamiento
  • Pila completa (Full-stack): Capacidad de trabajar en todas las capas de una aplicación

Es relativamente fácil de aprender

JavaScript está diseñado para ser más amigable que otros lenguajes de programación. Es como comparar aprender a conducir un coche automático versus uno manual: JavaScript es el automático del mundo de la programación. Además, tiene muchísimas "herramientas prefabricadas" (llamadas bibliotecas y frameworks) que son como plantillas o moldes que facilitan el trabajo. Es como tener un kit de IKEA con instrucciones claras en lugar de tener que construir un mueble desde cero.

Funciona en cualquier navegador

¿Has notado que puedes abrir una página web tanto en Chrome como en Safari o Firefox y funciona igual? Eso es gracias a JavaScript. Es compatible con todos los navegadores principales, sin necesidad de instalar programas adicionales. Es como escribir una carta en español: cualquier persona que hable español podrá leerla, sin importar si vive en España, México o Argentina.

Cada vez es más rápido y potente

En los últimos años, JavaScript se ha vuelto mucho más rápido y eficiente. Antes solo se usaba para cosas sencillas, pero ahora puede manejar aplicaciones complejas como videojuegos online, aplicaciones de video en tiempo real, o sistemas que procesan miles de datos al mismo tiempo. Es como la evolución de los teléfonos: antes solo servían para llamar, ahora son computadoras en tu bolsillo.

Conclusión

JavaScript combina versatilidad técnica con facilidad de uso, lo que explica su adopción masiva. Para los desarrolladores, representa una herramienta que puede manejar todo el stack tecnológico, mientras que para las empresas significa eficiencia y agilidad en sus proyectos de transformación digital.

2. ¿Cuáles son algunos tipos de datos JS?

En JavaScript, los tipos de datos se dividen en primitivos y de referencia . Te lo explico de forma clara y con ejemplos para que lo puedas usar en tu trabajo.

Tipos de datos primitivos:

  • String → cadenas de texto ("Hola")
  • Number → números enteros o decimales (42, 3.14)
  • Boolean → valores lógicos (true o false)
  • Undefined → variable declarada pero sin valor asignado
  • Null → valor intencionalmente vacío
  • Symbol → identificadores únicos (ES6)
  • BigInt → números enteros muy grandes (ES2020)

Tipos de datos de referencia:

  • Object → colecciones de datos ({ clave: valor })
  • Array → listas ordenadas ([1, 2, 3])
  • Function → bloques de código reutilizables
  • Otros: Date, Map, Set, etc.

// Ejemplo de varios tipos de datos en JS
let nombre = "Ana";           // String
let edad = 25;                // Number
let esMayor = true;           // Boolean
let dato;                     // Undefined
let vacio = null;             // Null
let id = Symbol("id");        // Symbol
let numeroGrande = 123n;      // BigInt
let persona = { nombre: "Ana", edad: 25 }; // Object

3. Métodos fundamentales para trabajar con Strings en JavaScript

Manipular texto es una de las tareas más comunes en programación web. Aquí veremos tres herramientas esenciales: .length, .substring() y .toLowerCase() / .toUpperCase().

1. .length – Contar caracteres

Es como usar una regla para medir la longitud de una palabra o frase. Cuenta todos los caracteres: letras, espacios, números y símbolos. 


// Ejemplo .length
let nombre = "JavaScript";
console.log(nombre.length); // 10

let frase = "Hola Mundo";
console.log(frase.length); // 10 (incluye el espacio)

Uso: Validar contraseñas, limitar caracteres en tweets o formularios.

2. .substring() – Cortar pedazos de texto

Es como usar unas tijeras para extraer la parte que necesitas. Se indica desde dónde empezar y dónde terminar (el final no se incluye).

  • Los índices empiezan en 0
  • substring(inicio, fin) → el final no se incluye
  • Si omites el segundo parámetro, corta hasta el final

// Ejemplo .substring()
let email = "usuario@ejemplo.com";
let nombreUsuario = email.substring(0, 7); // "usuario"
let dominio = email.substring(8); // "ejemplo.com"

let fecha = "2024-03-15";
let año = fecha.substring(0, 4); // "2024"
let mes = fecha.substring(5, 7); // "03"

3. .toLowerCase() y .toUpperCase() – Cambiar mayúsculas/minúsculas

Son como un interruptor para convertir texto a todo minúsculas o todo mayúsculas. Muy útiles para comparar y normalizar datos. 


// Ejemplo .toLowerCase() y .toUpperCase()
let nombre = "MARÍA GONZÁLEZ";
let nombreNormalizado = nombre.toLowerCase(); // "maría gonzález"

let grito = "ayuda";
let gritoFuerte = grito.toUpperCase(); // "AYUDA"

// Comparación sin importar mayúsculas
let respuesta1 = "SI";
let respuesta2 = "si";
if (respuesta1.toLowerCase() === respuesta2.toLowerCase()) {
    console.log("Las respuestas son iguales");
}

Casos prácticos combinando estos métodos:

  1. Validación de contraseñas: 
    
let contraseña = document.getElementById("password").value;
if (contraseña.length < 8) {
    alert("La contraseña debe tener al menos 8 caracteres");
}
  2. Procesar emails: 
    
let email = "Usuario@EJEMPLO.COM";
let emailLimpio = email.toLowerCase(); // "usuario@ejemplo.com"
let usuario = emailLimpio.substring(0, emailLimpio.indexOf("@"));
  3. Formatear nombres: 
    
let nombreCompleto = "JUAN PÉREZ LÓPEZ";
let apellido = nombreCompleto.substring(5, 10); // "PÉREZ"
let apellidoFormateado = apellido.toLowerCase(); // "pérez"

En resumen: Estos tres métodos son como las herramientas básicas de un carpintero: simples individualmente, pero poderosas al combinarlas.

4. Condicionales en JavaScript: Tomando Decisiones

En términos simples: Los condicionales son como los semáforos del código. Permiten que tu programa tome diferentes caminos según las circunstancias.

Técnicamente: Son estructuras de control que evalúan una condición (verdadero o falso) y ejecutan diferentes bloques de código según el resultado.

Analogía de la vida real: Es como cuando decides qué ropa ponerte. Si llueve, te pones impermeable. Si hace sol, te pones camiseta. Si hace frío, te pones abrigo. Los condicionales funcionan exactamente igual.

1. El condicional if - "Si esto, entonces aquello"

La estructura condicional más básica es if (si). Evalúa si algo es verdadero, y si lo es, ejecuta el código dentro de las llaves { }.


if (condición) {
    // Código que se ejecuta si la condición es verdadera
}

Ejemplo práctico: Control de acceso

Imagina un control de acceso a un club nocturno:


     
let edad = 20;

if (edad >= 18) {
    console.log("Puede entrar al club");
}

2. if-else - "Si no, entonces esto otro"

Muchas veces necesitas hacer algo cuando la condición NO se cumple. Para eso usamos else (si no).

   
if (condición) {
    // Se ejecuta si es verdadero
} else {
    // Se ejecuta si es falso
}

Ejemplo: Decidir actividades según el clima

        
let clima = "soleado";

if (clima === "lluvioso") {
    console.log("Quedarse en casa viendo Netflix");
} else {
    console.log("Salir a caminar al parque");
}

3. else if - "Si no esto, prueba esto otro"

Cuando tienes múltiples opciones, usas else if. Es como tener varias puertas y probar una por una hasta encontrar la correcta.

    
if (condición1) {
    // Opción 1
} else if (condición2) {
    // Opción 2
} else if (condición3) {
    // Opción 3
} else {
    // Opción por defecto
}

Ejemplo: Sistema de calificaciones


let puntuacion = 85;

if (puntuacion >= 90) {
    console.log("Excelente - Sobresaliente");
} else if (puntuacion >= 80) {
    console.log("Muy bien - Notable");
} else if (puntuacion >= 70) {
    console.log("Bien - Aprobado");
} else {
    console.log("Necesitas estudiar más");
}

4. Operadores de Comparación

Para crear condiciones, necesitas operadores condicionales

  • === - Exactamente igual
  • !== - Diferente
  • > - Mayor que
  • << /code> - Menor que
  • >= - Mayor o igual
  • <=< /code> - Menor o igual

Para combinar o modificar varias condiciones dentro de un mismo if, usamos operadores lógicos:

  • && (AND / "y") → Todas las condiciones deben ser verdaderas
  • || (OR / "o") → Basta con que una condición sea verdadera
  • ! (NOT / "no") → Invierte el valor lógico

Ejemplo: Sistema de descuentos en tienda online

       
let totalCompra = 120;
let esClienteVIP = true;

if (totalCompra >= 100 && esClienteVIP === true) {
    console.log("Descuento del 20%! Total: " + (totalCompra * 0.8) + "€");
} else if (totalCompra >= 100) {
    console.log("Descuento del 10%! Total: " + (totalCompra * 0.9) + "€");
} else {
    console.log("Total: " + totalCompra + "€");
}

Este ejemplo combina múltiples ramas (if, else if, else) y operadores lógicos.

  • En el primer if usamos && para comprobar que dos condiciones se cumplan al mismo tiempo:
  • El total sea mayor o igual a 100
  • El cliente sea VIP

Si una de esas dos condiciones no se cumple, el bloque no se ejecuta y se pasa a la siguiente rama.

5. Condicionales Anidados

Puedes poner condicionales dentro de otros condicionales, como muñecas rusas.

Ejemplo: Sistema de seguridad del coche


let tieneLicencia = true;
let edad = 25;
let estaEbrio = false;

if (tieneLicencia === true) {
    if (edad >= 18) {
        if (estaEbrio === false) {
            console.log("Puede conducir!");
        } else {
            console.log("No puede conducir - Estado no apto");
        }
    } else {
        console.log("Muy joven para conducir");
    }
} else {
    console.log("Necesita licencia de conducir");
}

Conceptos clave aprendidos:

  • Estructura condicional: Control de flujo del programa
  • Condición: Expresión que evalúa verdadero o falso
  • Operadores de comparación: Herramientas para crear condiciones
  • Bloques de código: Instrucciones dentro de llaves { }
  • Anidación: Condicionales dentro de otros condicionales

Resumen con analogía

Los condicionales son como ser un detective:

  • IF = "Si encuentro esta pista..."
  • ELSE IF = "Si no, pero encuentro esta otra..."
  • ELSE = "Si no encuentro nada de eso..."

Cada pista te lleva a una conclusión diferente, igual que cada condición lleva a ejecutar código diferente.

5. ¿Qué es un operador ternario?

El operador ternario es una forma concisa de escribir una declaración if-else en una sola línea. Se compone de tres partes:

  • La condición a evaluar (condición booleana)
  • El valor si la condición es verdadera (valor verdadero)
  • El valor si la condición es falsa (valor falso)

El operador ternario en Java es una construcción que permite tomar decisiones basadas en una condición booleana de manera concisa en una sola línea de código. Una condición booleana es una pregunta cuya respuesta solo puede ser verdadero (true) o falso (false). Es como un interruptor de luz: o está encendido (true) o está apagado (false), no hay términos medios.

Es una forma abreviada de expresar una estructura condicional (if-else).

Este operador se compone de tres partes: la condición a evaluar, el valor que se asigna si la condición es verdadera y el valor que se asigna si la condición es falsa.

La sintaxis que utiziarimos es la siguiente:


   condición ? valor_si_verdadero : valor_si_falso;

Por ejemplo en java: 


int edad = 20;
String mensaje = (edad >= 18) ? "Eres mayor de edad" : "Eres menor de edad";
System.out.println(mensaje);
  • condición → edad >= 18
  • valor_si_verdadero → "Puede entrar"
  • valor_si_falso → "No puede entrar"

La elección entre el operador ternario y la declaración if-else depende principalmente de la complejidad y la legibilidad del código que estés escribiendo.

El operador ternario es útil para simplificar la escritura de expresiones condicionales simples. Y es que, aunque el operador ternario en Java permite escribir código más compacto, en algunos casos puede afectar la legibilidad si se usa en expresiones muy largas. Para mejorar la organización del código, una buena práctica es encapsular la lógica dentro de una función.

6. Declaración de Función vs Expresión de Función

En JavaScript podemos crear funciones de dos formas principales: Declaración de Función y Expresión de Función. Ambas sirven para hacer lo mismo (crear un bloque de código que podemos reutilizar), pero funcionan de forma distinta.

1. Declaración de Función

Escribir una función como una “instrucción independiente” en el código. El motor de JavaScript la “prepara” antes de que el código empiece a ejecutarse, por lo que se puede usar incluso antes de escribirla en el archivo.

Ejemplo cotidiano:

Es como tener el teléfono de un amigo ya guardado en tu agenda: puedes llamarle cuando quieras, incluso antes de pensar en ello.


// Declaración de función
saludar("Ana"); // Funciona aunque esté antes de la definición

function saludar(nombre) {
    console.log(`Hola, ${nombre}`);
}

2. Expresión de Función

Es cuando guardamos una función dentro de una variable u otra estructura. La función solo existe a partir del momento en que el código llega a esa línea.

Ejemplo cotidiano:

Es como apuntar el teléfono de alguien en un papel justo cuando lo necesitas. Hasta que no lo escribes, no puedes usarlo.


// Expresión de función
saludar("Ana"); // Error: aún no existe

let saludar = function(nombre) {
    console.log(`Hola, ${nombre}`);
};

saludar("Ana"); // Funciona

Diferencia de alcance (scope)

En modo estricto, si una Declaración de Función está dentro de un bloque (por ejemplo, dentro de un if), solo se puede usar dentro de ese bloque.


let edad = 20;

if (edad < 18) {
    function bienvenida() {
        console.log("Hola joven");
    }
} else {
    function bienvenida() {
        console.log("Bienvenido adulto");
    }
}

// Fuera del bloque, esto dará error:
bienvenida(); // Error: no está definida

Solución con Expresión de Función

Podemos declarar la variable fuera del bloque y luego asignarle la función dentro.


let edad = 20;
let bienvenida;

if (edad < 18) {
    bienvenida = function() {
        console.log("Hola joven");
    };
} else {
    bienvenida = function() {
        console.log("Bienvenido adulto");
    };
}

bienvenida(); // Funciona

Resumen

  • Declaración de Función: puede usarse antes de escribirla, gracias al hoisting.
  • Expresión de Función: solo existe después de llegar a esa línea de código.
  • Usa Declaración de Función como primera opción, salvo que necesites declarar funciones de forma condicional o con un alcance más controlado.

7. ¿Qué es la palabra clave "this" en JS?

La palabra clave this en JavaScript

En JavaScript, this es una palabra clave que siempre hace referencia a un objeto. Ese objeto puede cambiar dependiendo de cómo y dónde se use.

Podemos imaginarlo como cuando decimos "yo" en la vida real: el significado de "yo" cambia según quién lo diga. En JavaScript, this funciona igual: su valor depende de quién "hable" (quién ejecute el código).

Casos prácticos en aplicaciones de oficina

¿Dónde usamos this en el mundo real? Aquí algunos ejemplos típicos:

1. Sistema de empleados:

let empleado = {
    nombre: "Juan Pérez",
    departamento: "Ventas",
    salario: 3000,
    
    calcularBonus: function() {
        return this.salario * 0.1; // 10% del salario
    },
    
    mostrarInfo: function() {
        return `${this.nombre} trabaja en ${this.departamento}`;
    }
};

console.log(empleado.calcularBonus()); // 300
console.log(empleado.mostrarInfo()); // "Juan Pérez trabaja en Ventas"

2. Carrito de compras:

let carrito = {
    productos: [],
    total: 0,
    
    agregarProducto: function(producto, precio) {
        this.productos.push({nombre: producto, precio: precio});
        this.total += precio;
    },
    
    mostrarResumen: function() {
        return `Tienes ${this.productos.length} productos por €${this.total}`;
    }
};

carrito.agregarProducto("Portátil", 800);
carrito.agregarProducto("Ratón", 25);
console.log(carrito.mostrarResumen()); // "Tienes 2 productos por €825"

3. Formularios dinámicos:

// En una página web, cada botón conoce sus propios datos
document.querySelectorAll('.boton-departamento').forEach(boton => {
    boton.addEventListener('click', function() {
        // 'this' se refiere al botón que fue clickeado
        let departamento = this.dataset.departamento;
        mostrarEmpleados(departamento);
    });
});

4. Creación de múltiples facturas:

function Factura(cliente, numero) {
    this.cliente = cliente;
    this.numero = numero;
    this.productos = [];
    this.fecha = new Date();
    
    this.agregarLinea = function(producto, cantidad, precio) {
        this.productos.push({
            producto: producto,
            cantidad: cantidad, 
            precio: precio,
            subtotal: cantidad * precio
        });
    };
    
    this.calcularTotal = function() {
        return this.productos.reduce((total, linea) => {
            return total + linea.subtotal;
        }, 0);
    };
}

// Crear facturas para diferentes clientes
let factura1 = new Factura("Empresa ABC", "F-001");
let factura2 = new Factura("Empresa XYZ", "F-002");

factura1.agregarLinea("Consultoría", 5, 100);
factura2.agregarLinea("Software", 1, 500);

console.log(factura1.calcularTotal()); // 500
console.log(factura2.calcularTotal()); // 500

¿Qué nos aporta this?

  • Reutilización: El mismo código funciona para diferentes objetos
  • Organización: Cada objeto mantiene sus propios datos y métodos
  • Escalabilidad: Podemos crear fácilmente múltiples instancias (empleados, facturas, etc.)
  • Mantenimiento: Los cambios en un objeto no afectan a otros

1. En el contexto global

Si usamos this fuera de cualquier función o clase, hace referencia al objeto global del entorno:

console.log(this); 
// En un navegador: window
// En Node.js: global

Modo estricto: Cuando escribes JavaScript en modo estricto ("use strict";), this será undefined en lugar del objeto global.

Ejemplo de la vida real: Es como estar en una sala grande y decir "yo estoy aquí". Como no perteneces a ningún grupo específico, te refieres a ti mismo en el contexto general de toda la sala.

2. Dentro de un objeto (método)

Cuando usamos this dentro de un método de un objeto, hace referencia al objeto que contiene ese método:

let mascota = {
    nombre: "Pintto",
    tipo: "perro",
    presentarse: function() {
        console.log(`Hola, soy ${this.nombre} y soy un ${this.tipo}`);
    }
};

mascota.presentarse(); 
// Resultado: "Hola, soy Pintto y soy un perro"

Ejemplo de la vida real: Si Pintto pudiera hablar y dijera "yo ladro", todos entenderían que "yo" se refiere a Pintto, no a su dueño.

3. Dentro de una función normal

En funciones normales (que no son métodos de un objeto), this apunta al objeto global:

function saludar() {
    console.log("Hola desde:", this);
}

saludar(); 
// Muestra el objeto global (window en navegador)

Ejemplo de la vida real: Es como una persona que trabaja de forma independiente (freelancer). Cuando dice "yo trabajo", no representa a ninguna empresa específica, actúa por su cuenta.

4. En constructores y clases

Cuando usamos new para crear objetos, this hace referencia al nuevo objeto que se está creando:

function Auto(marca, modelo) {
    this.marca = marca;
    this.modelo = modelo;
    this.describir = function() {
        return `Soy un ${this.marca} ${this.modelo}`;
    };
}

let miAuto = new Auto("Toyota", "Corolla");
console.log(miAuto.describir()); 
// "Soy un Toyota Corolla"

Ejemplo de la vida real: Es como cuando nace un bebé. Al momento de nacer, cuando eventualmente diga "yo", se referirá a sí mismo, a su nueva identidad individual.

5. Con funciones flecha

Las funciones flecha son especiales: NO tienen su propio this. En su lugar, toman el this del lugar donde fueron definidas (no donde se ejecutan).

Veamos la diferencia:

let persona = {
    nombre: "María",
    saludar: function() {
        console.log(`Función normal: ${this.nombre}`); // María ✓
    },
    despedirse: () => {
        console.log(`Función flecha: ${this.nombre}`); // undefined ✗
    }
};

persona.saludar();    // "Función normal: María"
persona.despedirse(); // "Función flecha: undefined"

¿Por qué pasa esto?

  • La función normal (saludar) crea su propio this que apunta al objeto persona
  • La función flecha (despedirse) fue creada en el contexto global, entonces usa el this global (que no tiene propiedad nombre)

Un ejemplo más claro de cuándo sí funciona la función flecha:

let persona = {
    nombre: "María",
    saludarTodos: function() {
        let amigos = ["Ana", "Luis"];
        
        // Función flecha DENTRO de un método
        amigos.forEach(amigo => {
            console.log(`${this.nombre} saluda a ${amigo}`);
            // Aquí this.nombre SÍ funciona porque hereda 
            // el 'this' de saludarTodos()
        });
    }
};

persona.saludarTodos();
// "María saluda a Ana"
// "María saluda a Luis"

Ejemplo de la vida real: Una función flecha es como un asistente que no tiene identidad propia: siempre habla en nombre de su jefe. Si el asistente trabaja para el director, dirá "yo (el director) apruebo esto". Si trabaja de forma independiente, no puede decir "yo apruebo" porque no tiene autoridad propia.

Resumen

  • this es como decir "yo" en programación
  • En métodos de objetos: "yo" soy el objeto que me contiene
  • En funciones normales: "yo" soy el contexto global
  • Con new: "yo" soy el nuevo objeto que se está creando
  • En funciones flecha: "yo" soy heredado de donde fui creada

Consejo práctico: Cuando te confundas con this, pregúntate: "¿Quién está ejecutando este código en este momento?" Esa será tu respuesta.
